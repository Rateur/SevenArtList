# Contributing to SevenArtList

First off, thank you for considering contributing to SevenArtList! It's people like you who make this tool better for everyone.

## Code of Conduct

Help us keep SevenArtList open and inclusive. Please read and follow our [Code of Conduct](CODE_OF_CONDUCT.md) (coming soon).

## How Can I Contribute?

### Reporting Bugs

If you find a bug, please use the GitHub Issues tab to report it. Provide as much detail as possible, including:
- Steps to reproduce the bug.
- Expected behavior.
- Actual behavior.
- Screenshots if applicable.

### Suggesting Enhancements

We're always looking for ways to improve! Please open an issue to discuss new features or improvements.

### Pull Requests

1. **Fork** the repository.
2. Create a new branch for your work:
   - For new features: `feat/your-feature-name`
   - For bug fixes: `fix/your-bug-name`
3. Make your changes.
4. Use **Conventional Commits** for your commit messages (e.g., `feat(auth): add login button`, `fix(ui): fix responsive navbar`).
5. Ensure your code follows the project's coding standards (run `npm run lint`).
6. Submit a **Pull Request** to the `main` branch.
